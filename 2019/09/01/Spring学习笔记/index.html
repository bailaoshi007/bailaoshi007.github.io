Spring

Spring 春天

Spring 是开源组织。Spring组织提供了很多开源组件；

Spring 两大核心功能：IOC/DI和AOP

IOC

IOC控制反转：指对象的创建和管理

  1. 主动控制：由应用程序创建和管理对象，称为主动控制
    • 适合创建简单对象
  2. 控制反转：应用程序将对象的创建和管理交给环境处理，应用程序只是被动的使用对象，称为控制反转。
    • 适合创建和管理复杂对象

JavaBean

JavaBean是指其类型满足一定规则的java对象。规范如下

  1. ==类必须==有包（package）；==//有包名才可以导包，然后调用.class==
  2. ==类必须==有无参构造器，默认构造器就是无参构造器；
    • 方便派生子类，不理解==？？？？？？==
  3. 实现序列化接口，方便java底层自动进行对象序列化；==测试发现不实现序列化接口也是可以的==
  4. ==类必须==是public修饰的；
  5. ==可以==包含Bean属性。Bean属性就是setXxx、getXxx方法。
    • 当有方法getName，setName时，就称为包含Bean属性name；
    • boolean类型的Bean属性方法：getXxx或isXxx，==没有setXxx==
  6. ==必须的构成一句口诀：包公无（包：必须有包；公：public；无：无参构造）==

例子：

public boolean setMarried(){
return married;
}这种写法不对

Spring IOC

Spring IOC是管理JavaBean对象的容器。

  1. Spring 建议被管理的对象采用JavaBean规范。
    • Spring可以宽泛的支持Java对象，几遍没有遵守JavaBean规范
  2. JavaBean规范不是强制性规范，但是被所有企业作为变成规范。
